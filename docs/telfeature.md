Prompt for your agent:
You are an expert full-stack developer specializing in Cloudflare Pages + Functions.
Project Goal: Implement a protected checklist website on Cloudflare Pages where:
	•	The page is protected by an access code generated by a Telegram bot.
	•	When a user checks a task checkbox, it sends a formatted message to a specific Telegram group.
	•	All secrets (BOT_TOKEN, CHAT_ID) must remain completely hidden from the browser.
	•	Use Cloudflare best practices (Pages Functions for backend logic, KV for code storage if needed).
Current State: The site already exists but does not yet have the checklist + Telegram + protection functionality. You will add this on top of the existing site.
Requirements:
	1	Access Protection
	◦	User must first obtain a code by messaging the Telegram bot (/start or /code).
	◦	On the website, show a login screen asking for the 8-character code.
	◦	Validate the code via a Cloudflare Function.
	◦	On success, hide the login screen and show the checklist. Persist access in localStorage for the session.
	◦	Codes should be temporary (expire after 1 hour) and preferably one-time use.
	2	Checklist Functionality
	◦	Multiple checkboxes with task names (use data-task attribute).
	◦	When a checkbox is checked (but not when unchecked), immediately send the task name to the Telegram group.
	◦	Message format example: ✅ Task completed: *Buy groceries*
	3	Telegram Integration
	◦	Use Telegram Bot API (sendMessage).
	◦	Support Markdown.
	◦	Secrets must be read from Cloudflare environment variables only.
	4	Cloudflare Architecture (Mandatory)
	◦	Static files in /public (or your existing static folder).
	◦	Backend logic in /functions/api/:
	▪	verify-code.js (POST) — validates access code.
	▪	check-task.js (POST) — sends message to Telegram.
	◦	Store temporary codes in a Cloudflare KV namespace called CODES.
	◦	Bind the KV namespace in Pages settings.
	◦	All secrets (BOT_TOKEN, CHAT_ID) set as Secret environment variables in Pages dashboard (Production + Preview).
	5	Security
	◦	Never expose any secret in frontend JavaScript or HTML.
	◦	Validate inputs.
	◦	Return proper HTTP status codes.
	6	Deliverables
	◦	Updated folder structure with all new files.
	◦	Clear instructions on how to:
	▪	Set up the KV namespace.
	▪	Add environment variables/secrets in Cloudflare Pages.
	▪	Deploy.
	▪	Test locally with wrangler pages dev.
	◦	Minimal, clean, production-ready code.
	◦	Make the UI clean and mobile-friendly (you may use Tailwind if it fits easily).
Existing Tech Stack: Cloudflare Pages (confirm any framework if present, otherwise assume vanilla HTML/JS).
Provide the full code for each file and step-by-step deployment instructions at the end.
